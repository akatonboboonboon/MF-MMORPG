## 1. この文書でいう「確定」

この文書には、参照会話でユーザーが明示した内容と、今回の依頼で指定された内容だけを記載する。過去のアシスタントによる提案は、ユーザーが明示的に追認していない限り確定仕様に含めない。

情報が競合する場合の優先順位は次のとおりとする。

1. 今回の依頼に含まれる明示的な数値・範囲・禁止事項
2. 会話内でユーザーが「決定」として述べた事項
3. ユーザーが方向性として述べた事項
4. アシスタントの提案。これは候補に留まり、確定事項を上書きしない

各主要仕様には、決定根拠を追跡するため次の `origin` を付ける。

| origin | 意味 |
|---|---|
| `user_explicit` | ユーザーが直接、決定または制約として明示した内容 |
| `user_approved` | アシスタントまたはCodexの提案を、ユーザーが後から明示的に承認した内容 |
| `assistant_proposal` | 参照会話内のアシスタント提案。未承認なら候補に留める |
| `codex_recommendation` | 文書化・実装可能化のためにCodexが提示した推奨。未承認なら候補に留める |

今回の改訂依頼で直接指定された変更は `user_explicit` とする。出所は科学的正しさや実装優先度ではなく、「誰がどの段階で決めたか」を示す。

参照会話には複数の添付画像が存在する。画像内容がCodexの作業環境へ明示的に渡されていない場合、画像だけに含まれる情報は確定仕様へ自動採用しない。

### 3.1 プレイヤーを構成する確定5層モデル

プレイヤーデータは次の5層へ分離する。この境界は今回の改訂で解決済みとし、単一の「素材キャラクター」クラスへ押し込まない。

| 層 | 保持するもの | 戦闘上の責務 | origin |
|---|---|---|---|
| `AvatarIdentity` | 名前、顔、髪、声、人格など | 戦闘性能へ影響しない恒久的な本人情報 | `user_explicit` |
| `MaterialJob` | ナイト、アルミ、銅、SUSなどの材料系統、基本物性、魔法適性、材料スキルツリー | **性能と壊れ方**を決める | `user_explicit` |
| `CombatForm` | 刃、板、盾、線、管、容器などの製品形状 | **操作方法、移動補正、アクションセット、攻撃方法**を決める | `user_explicit` |
| `EquipmentInstance` | 現在使う武器・鎧の個体、耐久、変形、加工状態、疲労履歴、個体スキルツリー | 個体単位の損傷、加工、疲労破壊を保持する | `user_explicit` |
| `Appearance` | 鎧・武器の主色、副色、発光色、外装 | 性能と分離された表示を決める | `user_explicit` |

`MaterialJob` と `CombatForm` は独立させる。同じ材料でも形状が違えば操作と攻撃方法が変わり、同じ形状でも材料が違えば性能と壊れ方が変わる。

### 3.2 EquipmentLoadoutと最小試作の損傷所有権

`EquipmentLoadout` は5層へ新たな第6層を足すものではなく、複数の `EquipmentInstance` を束ねる集約概念とする。

| ID | 確定事項 | origin |
|---|---|---|
| CF-016 | `EquipmentLoadout` は `core_equipment_instance_id` と `equipment_instance_ids[]` を持つ。 | `user_explicit` |
| CF-017 | 最小試作ではcore装備1個だけを使用し、そのcore装備がプレイヤーの `Integrity`、`Deformation`、`temperature` を所有する。独立した武器耐久、鎧耐久、複数装備間の損傷分配は後回しにする。 | `user_explicit` |
| CF-018 | 最小試作の `MaterialJob` と `CombatForm` の選択はcore装備側を権威元とする。Actor側に同じ参照を置く場合は読み取り専用の別名またはキャッシュとし、独立して書き換え可能な二重の正を作らない。 | `user_explicit` |

## 12. 承認ゲート

**Gate 0は閉じたままである。** この改訂自体は実装開始の承認ではない。

この文書を含む10文書をユーザーが確認し、少なくとも実装開始に必要な未確定事項を決定するまで、次を行わない。

- ゲームエンジンの本プロジェクト作成
- ゲーム本体コードの大規模実装
- 本番用アート・音声の量産
- ネットワーク、アカウント、課金の実装
